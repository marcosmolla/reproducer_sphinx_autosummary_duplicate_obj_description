# Sphinx autodoc: duplicate object warnings with generic pydantic BaseModel

Minimal reproducer for "duplicate object description" warnings when `sphinx.ext.autodoc`
documents a generic `pydantic.BaseModel` that has parametrized subclasses.

## Requirements

- Python 3.13+
- [uv](https://docs.astral.sh/uv/)

## Reproduce

```bash
./reproduce.sh
```

This will:
1. Install dependencies via `uv sync`
2. Run `sphinx-apidoc` to generate `.rst` files
3. Run `sphinx-build -W` (warnings as errors)

## Expected

Build succeeds without warnings.

## Actual

```
<unknown>:1: WARNING: duplicate object description of mypackage.base.IBase, other instance in generated/mypackage.base, use :no-index: for one of them
<unknown>:1: WARNING: duplicate object description of mypackage.base.IBase.model_config, ...
<unknown>:1: WARNING: duplicate object description of mypackage.base.IBase.create_state, ...
```

## Verified conditions (ALL required to reproduce)

1. `sphinx.ext.autosummary` in extensions (even though no `.. autosummary::` directive is used)
2. `pydantic.BaseModel` as the base class
3. Generic type parameters (PEP 695 or old-style `Generic[T]` — both trigger it)
4. A subclass that parametrizes the generic (`class Child(IBase[ConcreteState])`)
5. Both parent and child documented via `automodule` (as generated by `sphinx-apidoc`)

## Does NOT happen when

- `sphinx.ext.autosummary` is removed from extensions → no warning
- `pydantic.BaseModel` is replaced with a plain class → no warning
- Inheritance is unparametrized (`class Child(IBase)`) → no warning

## Notes

- In larger codebases, the same "duplicate object description" warnings can also appear
  *without* `autosummary` — suggesting the root cause is in autodoc's object registration
  logic, and `autosummary` is one (but not the only) code path that exposes it.
- `sphinx_autodoc_typehints` is NOT required to reproduce.
- `:show-inheritance:` is NOT required to reproduce.
- `--separate` flag to sphinx-apidoc is NOT required to reproduce.

## Versions

- Python 3.13
- Sphinx 9.1.0
- pydantic 2.x
